many diffrent JavaScript security issues have gained attention. one such example is, the way JavaScript interacts with the DOM poses a risk for end users by enabling malicious actors to send scripts over the web and run them on other computers. There are two measures that can be taken to block this JavaScript security risk. First is known as sandboxing which is where you run code, observe and analyze the code in a safe environment on a network that mimics client user operating environments. The second measure is implementing the same origin policy, which stops scripts from a site from accessing data that is used by scripts form another site.
